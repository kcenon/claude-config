# Incident Report — 2026-05-01

## Summary

During the first execution of `create-issues.sh --execute`, a stale `id-map.json` file
left over from earlier dry-run testing caused the registration script to skip PASS 1
(creation) and proceed directly to PASS 2 (placeholder substitution) using the mock
numbers `#100`–`#129`. Those numbers correspond to **already-existing closed issues
and merged PRs** in this repository, so PASS 2 overwrote 30 historical bodies with
memory-sync issue content.

## Root cause

`create-issues.sh` PASS 1 uses `id_get` to check whether an ID has been assigned.
If the ID is already mapped (regardless of whether the mapped number is real or
mock), the issue is skipped. The dry-run test path called `save_id_map` to persist
the mock mapping for cross-pass continuity within the dry-run, and that file was
not deleted before the production run.

## Mitigations applied

- All 30 overwritten bodies restored from GitHub's `userContentEdits` history
  (verified via spot-check on `#100`, `#115`, `#129`)
- 30 new memory-sync issues created at `#505`–`#534`; placeholder substitution
  applied correctly
- `id-map.json` for the real run committed at `scripts/memory-sync-issues/tooling/id-map.json`
  for posterity

## Future prevention

Recommended changes to `create-issues.sh` (filed as a follow-up task — not done in
this incident response):

1. **Refuse to run if `id-map.json` is older than 24 hours** without explicit
   `--resume` flag — stale dry-run data should not silently influence production runs.
2. **Print the loaded id-map before PASS 1** when running with `--execute` and
   require user confirmation if any mapped value matches the format of a mock placeholder
   (e.g., starts with `DRY-` or is suspiciously sequential like 100..129).
3. **Decouple dry-run state from execute state** — write dry-run map to
   `id-map.dry-run.json` and execute map to `id-map.json`.
4. **Validate that target IDs do not already exist** before PASS 2 edits — query
   `gh issue view <N>` and refuse if the title doesn't match the expected new issue
   title.
