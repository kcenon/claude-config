## What

Create the `kcenon/claude-memory` private GitHub repository, seed it with directory skeleton, baseline memory files (post-#B3 trust-classified), and the validation tooling moved from interim location.

### Scope (in)

- New private GitHub repo under @kcenon
- Initial directory layout: `memories/`, `quarantine/`, `audit/`, `archive/`, `scripts/`, `tests/`
- Seed `memories/` with the 17 baseline memories (with #B2 backfilled fields and #B3 confirmed tiers)
- Move `validate.sh`, `secret-check.sh`, `injection-check.sh`, `quarantine-move.sh`, `quarantine-restore.sh`, `backfill-frontmatter.sh` from interim location to this repo
- Initial README.md with cross-link to `kcenon/claude-config/docs/MEMORY_SYNC.md`
- `.gitignore`, `.gitattributes` for LF + UTF-8 enforcement
- Branch protection rules on `main`

### Scope (out)

- pre-commit hook installation (#C3)
- Commit signing setup per machine (#C4)
- GitHub Actions workflow (#C5)
- Index generator (#C2 — though directory must exist)

## Why

Phase A and B produce tools and decisions, but they live in temporary locations (`/tmp/claude/...` or `kcenon/claude-config/scripts/memory/`). They need a permanent home before further infrastructure (Phase C, D) can reference them by stable paths.

The repo also serves as the **first end-to-end smoke test**: if 17 memories with all corrected metadata and validators don't fit cleanly together, it surfaces here before sync code is written.

- #C2 — index generator references `memories/` directory

- #C4 — branch protection requires a repo to protect
- #C5 — GitHub Actions workflow lives in this repo
- #D1 — sync engine clones from this URL
- #E1 — migration runbook references this repo

### Edge Cases

- **Repo already exists** (re-running this issue) → script detects, refuses to overwrite
- **GH auth lacks repo creation scope** → fail loudly with link to GitHub token settings
- **Seed commit unsigned** because #C4 setup not done on the seeding machine → block; user must run #C4 first OR force unsigned with explicit `--unsigned` flag (not recommended)
- **`memories/` already populated incorrectly somehow** → script verifies file count == 17 before proceeding
- **Network interruption mid-push** → seed commit local; `git push` retried manually; this is a one-shot, not automated
- **Filename mismatch** (a baseline memory doesn't fit `<type>_<topic>.md` pattern post-#B3 changes) → block, user investigates
